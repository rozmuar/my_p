# Назначение Супер Админа в PostAPI

Существует **3 способа** стать супер админом в системе:

## 🚀 1. Автоматическое назначение первого пользователя

**КАК ЭТО РАБОТАЕТ:**
- Первый зарегистрированный в системе пользователь автоматически получает роль `superAdmin`
- Все остальные пользователи получают роль `user`
- Логика встроена в `users.service.ts`

**ПРИМЕР:** 
1. Очистите базу данных (удалите все записи из таблицы users)
2. Зарегистрируйтесь на http://62.113.110.83:8080/register
3. Автоматически получите права супер админа

---

## 🛠️ 2. Ручное назначение через базу данных

**ПОДКЛЮЧЕНИЕ К БД:**
```bash
# Подключение к PostgreSQL контейнеру
docker exec -it <postgres_container_id> psql -U postgres -d postapi
```

**SQL КОМАНДЫ:**
```sql
-- Посмотреть всех пользователей
SELECT id, name, email, role FROM users;

-- Назначить супер админа по email
UPDATE users 
SET role = 'superAdmin' 
WHERE email = 'your-email@example.com';

-- Проверить изменения
SELECT * FROM users WHERE role = 'superAdmin';
```

---

## 👑 3. Назначение через API (если есть доступ)

**ENDPОINT:** `PUT /api/admin/users/{id}/promote`

**ТРЕБОВАНИЯ:**
- Нужны права `superAdmin` для выполнения
- Только супер админ может создавать других супер админов

**ПРИМЕР запроса:**
```bash
curl -X PUT "http://62.113.110.83:3001/api/admin/users/123/promote" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

**Ответ:**
```json
{
  "message": "Пользователь Иван Петров назначен супер админом",
  "user": {
    "id": "123",
    "email": "ivan@example.com", 
    "name": "Иван Петров",
    "role": "superAdmin"
  }
}
```

## 🔐 Права супер админа

**ДОСТУП К:**
- ✅ Административная панель (`/admin`)
- ✅ Управление пользователями 
- ✅ Назначение ролей другим пользователям
- ✅ Управление новостями
- ✅ Полный доступ к API
- ✅ Удаление пользователей
- ✅ Промо других пользователей в админы

**ИЕРАРХИЯ РОЛЕЙ:**
1. `superAdmin` - полный доступ
2. `admin` - управление пользователями
3. `editor` - редактирование контента
4. `viewer` - только просмотр
5. `user` - базовые права

---

## 🚨 Безопасность

**РЕКОМЕНДАЦИИ:**
- Назначайте супер админа только проверенным людям
- Не раздавайте роль `superAdmin` массово
- Регулярно проверяйте список супер админов
- При смене команды отзывайте права у бывших сотрудников

**ПРОВЕРКА супер админов:**
```sql
SELECT name, email, "createdAt" 
FROM users 
WHERE role = 'superAdmin' 
ORDER BY "createdAt";
```
